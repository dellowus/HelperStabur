# Куда вставлять виджет в Tilda и как выложить backend

## Часть 1. Куда в Tilda вставлять код

### Шаг 1: Открыть настройки сайта

1. Зайдите в **Tilda** (tilda.cc) и откройте свой сайт **psve.ru** в редакторе.
2. Нажмите **«Настройки»** (иконка шестерёнки в левом меню или вверху).
3. В левом меню выберите **«Дополнительно»** или **«Расширенные настройки»**.

### Шаг 2: Найти блок с кодом перед `</body>`

В разделе «Дополнительно» найдите один из вариантов:

- **«Код перед закрывающим тегом `</body>`»**, или  
- **«Дополнительный код»** → вкладка **«В конец страницы»** / **«Before </body>»**.

Там есть большое текстовое поле — в него и нужно вставлять код.

### Шаг 3: Вставить одну строку

Вставьте **одну строку** (подставьте вместо `https://ВАШ-ХОСТИНГ` реальный адрес вашего backend — см. раздел про хостинг ниже):

```html
<script src="https://ВАШ-ХОСТИНГ/widget/embed.js" data-api="https://ВАШ-ХОСТИНГ"></script>
```

**Пример**, если backend будет на `https://my-helper.railway.app`:

```html
<script src="https://my-helper.railway.app/widget/embed.js" data-api="https://my-helper.railway.app"></script>
```

- Не удаляйте другой код, если он уже есть в этом блоке — просто добавьте эту строку **в конец**.
- Сохраните настройки и опубликуйте сайт.

После публикации на psve.ru в правом нижнем углу появится кнопка чата. Она будет работать только когда backend уже выложен и по адресу `https://ВАШ-ХОСТИНГ` открывается ваш сервер.

## Часть 2. Какой хостинг выбрать и как выложить backend

Backend — это папка **HelperStabur** с файлами `server.js`, `knowledge.js`, `widget/` и т.д. Его нужно запустить на сервере с **Node.js** и **HTTPS**.

### Варианты хостинга

| Хостинг | Плюсы | Минусы | Подходит |
|--------|--------|--------|----------|
| **Railway** | Бесплатный тариф, просто, автоматический HTTPS | Лимиты по трафику/времени | Да, для старта |
| **Render** | Бесплатный тариф, HTTPS из коробки | «Засыпает» после 15 мин без запросов | Да |
| **Timeweb, Reg.ru, Beget** (VPS/Node) | Российские, предсказуемая оплата | Нужно самому настраивать Node и SSL | Если уже есть сервер |
| **Vercel / Netlify** | Бесплатно | Нужно переписать backend под serverless | Только если готовы переделать код |

Ниже — пошагово два варианта: **Railway** (проще всего) и **VPS** (если хостинг уже есть).

## Вариант A: Деплой на Railway (рекомендуется для старта)

1. Зарегистрируйтесь на [railway.app](https://railway.app) (через GitHub удобнее).
2. Создайте новый проект → **Deploy from GitHub repo** (или загрузите папку HelperStabur как репозиторий).
3. В настройках сервиса добавьте **Variables** (переменные окружения):
   - `TELEGRAM_BOT_TOKEN` = `8750571471:AAHoOCjNw5f42tOJmqCXSZ-M2sHrsyxdBCs`
   - `TELEGRAM_CHAT_ID` = `@promauto_rf`
   - `ALLOWED_ORIGIN` = `https://psve.ru`
   - `PORT` Railway обычно подставляет сам; если просят — укажите `3000`.
4. Railway сам соберёт проект (нужен `package.json` и `npm start`), выдаст ссылку вида `https://ваш-проект.up.railway.app`.
5. Эту ссылку подставьте в Tilda вместо `https://ВАШ-ХОСТИНГ` в строке скрипта выше.

Итоговая строка для Tilda будет такой (с вашим URL от Railway):

```html
<script src="https://ваш-проект.up.railway.app/widget/embed.js" data-api="https://ваш-проект.up.railway.app"></script>
```

## Вариант B: Деплой на свой VPS (Timeweb, Reg.ru, Beget и т.д.)

Если у вас уже есть Linux-сервер (VPS):

1. Подключитесь по SSH.
2. Установите Node.js 18+ (например: `curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -` и `sudo apt install nodejs`).
3. Загрузите папку HelperStabur на сервер (git clone или через FTP/SFTP).
4. В папке проекта выполните:
   ```bash
   npm install
   ```
5. Создайте файл `.env` в этой же папке с содержимым:
   ```
   TELEGRAM_BOT_TOKEN=8750571471:AAHoOCjNw5f42tOJmqCXSZ-M2sHrsyxdBCs
   TELEGRAM_CHAT_ID=@promauto_rf
   ALLOWED_ORIGIN=https://psve.ru
   PORT=3000
   ```
6. Запустите в фоне (например через pm2):
   ```bash
   npm install -g pm2
   pm2 start server.js --name helperstabur
   pm2 save && pm2 startup
   ```
7. Настройте Nginx (или аналог) как прокси на `http://127.0.0.1:3000` и включите HTTPS (Let's Encrypt). Домен может быть любой, например `helper.psve.ru` или отдельный поддомен.
8. В Tilda в качестве `https://ВАШ-ХОСТИНГ` укажите этот домен (например `https://helper.psve.ru`).

## Проверка после деплоя

1. Откройте в браузере: `https://ВАШ-ХОСТИНГ/api/health`  
   Должно вернуться: `{"ok":true,"service":"helperstabur"}`.
2. Откройте psve.ru — в правом нижнем углу должна быть кнопка чата. Задайте тестовый вопрос — ответ должен прийти в чат, а сообщение — в канал **@promauto_rf**.

## Важно про безопасность

Токен бота вы однажды написали в чат. Если переживаете за утечку:

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram.
2. Отправьте `/mybots` → выберите бота → **API Token** → **Revoke current token**.
3. Скопируйте новый токен и подставьте его в `.env` на сервере и в переменных окружения хостинга (Railway/VPS), затем перезапустите backend.

Файл `.env` в папке проекта уже добавлен в `.gitignore` — в репозиторий он не попадёт. В Tilda вставляется только ссылка на ваш backend, **без токена**.
