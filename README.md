# Виртуальный помощник для psve.ru (ПЛК СТАБУР)

Виджет в правом нижнем углу сайта: отвечает на типовые вопросы по руководствам, прайсу и FAQ. Все вопросы дублируются в Telegram-канал. **Токен бота хранится только на сервере** — в браузере его нет, украсть через F12 невозможно.

## Что сделано

- **Backend (Node.js)**  
  - Принимает вопросы по API и отправляет их в Telegram (токен только в переменных окружения на сервере).  
  - Отдаёт ответы из базы знаний (FAQ, прайс, руководства).

- **Виджет для Tilda**  
  - Кнопка в правом нижнем углу, окно чата, стиль под psve.ru (синий/серый).

- **Безопасность**  
  - Токен Telegram не попадает в фронтенд. Запросы с сайта идут только на ваш backend; в Telegram пишет уже сервер.

## Быстрый старт

### 1. Установка и настройка

```bash
cd HelperStabur
npm install
cp .env.example .env
```

В `.env` укажите:

- `TELEGRAM_BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather).
- `TELEGRAM_CHAT_ID` — ID канала (например `@mychannel` или `-1001234567890`). Бот должен быть добавлен в канал как администратор.
- `ALLOWED_ORIGIN=https://psve.ru` (и при необходимости другие домены).
- `PORT=3000` (или порт вашего хостинга).

### 2. Запуск

```bash
npm start
```

Проверка: откройте `http://localhost:3000/api/health` — должен вернуться `{"ok":true}`.

### 3. Подключение виджета на Tilda

В Tilda: **Настройки сайта → Дополнительно → Код перед закрывающим тегом `</body>`** (или «Дополнительный код»).

Вставьте одну строку (подставьте вместо `https://ВАШ-СЕРВЕР.ru` адрес вашего backend):

```html
<script src="https://ВАШ-СЕРВЕР.ru/widget/embed.js" data-api="https://ВАШ-СЕРВЕР.ru"></script>
```

Важно: в виджете указан только `data-api` — **никакого токена**. Все запросы идут на ваш сервер, а сервер уже отправляет сообщения в Telegram.

### 4. Деплой backend

Backend нужно разместить на любом сервере с Node.js (VPS, облако, Railway, Render, Vercel Serverless и т.п.):

- Установите переменные окружения из `.env` в настройках хостинга.
- Убедитесь, что доступен HTTPS (для production лучше всегда использовать HTTPS).
- В `ALLOWED_ORIGIN` укажите `https://psve.ru` (и при необходимости `https://www.psve.ru` и домен Tilda, если запросы идут с него).

После деплоя замените в коде на Tilda `https://ВАШ-СЕРВЕР.ru` на реальный URL вашего backend.

## Расширение базы знаний

Файл `knowledge.js` содержит массив вопросов и ответов. Можно:

- Добавлять новые блоки с `keywords` и `answer`.
- Подставлять фразы из руководств по эксплуатации и актуального прайса.
- Позже заменить простой подбор по ключевым словам на поиск по тексту или запросы к LLM.

## Безопасность

- Токен бота — только в `.env` на сервере, в репозиторий не коммитить.
- CORS настроен на разрешённые домены (psve.ru и т.д.).
- Лимит размера тела запроса 100 KB.
- В браузере пользователь видит только обращение к вашему API; Telegram для него недоступен.
