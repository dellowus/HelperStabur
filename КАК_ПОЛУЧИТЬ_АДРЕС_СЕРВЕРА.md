# Как получить адрес для backend (простыми словами)

## В чём суть

- **Backend** — это ваша папка HelperStabur с программой `server.js`, которая принимает вопросы с сайта и шлёт их в Telegram.
- Сейчас она работает только у вас на компьютере (localhost). Сайт psve.ru в интернете до неё «не дотягивается».
- **Адрес сервера** — это ссылка вида `https://какой-то-сайт.ru` или `https://ваш-проект.up.railway.app`. Такой ссылки у вас **ещё нет** — её **даёт хостинг**, когда вы туда загружаете проект.
- То есть: вы не «узнаёте» готовый адрес, вы его **создаёте**, выложив backend на хостинг. Хостинг после публикации выдаёт вам ссылку — это и есть адрес вашего backend.

Дальше — самый простой способ получить такой адрес бесплатно (на примере Railway).

---

## Пошагово: получить адрес за 10–15 минут (Railway)

### Шаг 1. Регистрация на Railway

1. Откройте в браузере: **https://railway.app**
2. Нажмите **「Start a New Project」** или **「Login」**.
3. Войдите через **GitHub** (если нет аккаунта — зарегистрируйтесь на github.com, это бесплатно). Railway попросит разрешить доступ — нажмите «Authorize».

Адрес сервера у вас пока ещё нет — он появится в конце.

---

### Шаг 2. Создать репозиторий на GitHub (если ещё нет)

Чтобы Railway мог забрать ваш код, его нужно положить в репозиторий на GitHub:

1. Зайдите на **https://github.com** и войдите в аккаунт.
2. Нажмите **「+」** → **「New repository」**.
3. Имя репозитория: например **HelperStabur**.
4. Оставьте **Public**, галочку «Add a README» можно не ставить.
5. Нажмите **「Create repository」**.

Дальше — загрузить в этот репозиторий папку HelperStabur с вашего компьютера.

**Важно:** в репозиторий не должны попасть секреты. Файл **`.env`** в проекте уже в `.gitignore` — в GitHub он не загрузится. Токен бота потом добавите в Railway вручную.

**Как загрузить код в GitHub (с компьютера):**

1. Откройте папку `c:\ProjectAI\HelperStabur` в проводнике.
2. В адресной строке напишите `cmd` и нажмите Enter — откроется командная строка в этой папке.
3. Выполните по очереди (подставьте вместо `ВАШ_ЛОГИН` ваш логин на GitHub):

```bash
git init
git add .
git commit -m "First commit"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/HelperStabur.git
git push -u origin main
```

Если Git спросит логин/пароль — используйте логин GitHub и **Personal Access Token** вместо пароля (в GitHub: Settings → Developer settings → Personal access tokens → создать токен с правом repo).

**Если не хотите использовать GitHub:** в разделе ниже есть вариант «без GitHub» — загрузка проекта на Railway вручную (через архив).

---

### Шаг 3. Создать проект в Railway и получить адрес

1. На **railway.app** нажмите **「New Project」**.
2. Выберите **「Deploy from GitHub repo」**.
3. Если нужно — нажмите **「Configure GitHub App」** и выберите репозиторий **HelperStabur** (или дайте доступ ко всем репозиториям).
4. Выберите репозиторий **HelperStabur** и нажмите **「Deploy」**.

Railway начнёт сборку. Через 1–2 минуты появится надпись, что сервис запущен, но **публичной ссылки ещё не будет**.

5. Откройте созданный сервис (клик по нему).
6. Перейдите во вкладку **「Settings」**.
7. Найдите блок **「Networking」** → **「Generate Domain」** (или **「Public Networking」** → **「Generate domain」**). Нажмите.
8. Railway выдаст ссылку вида:  
   **`https://helperstabur-production-xxxx.up.railway.app`**  
   Это и есть **адрес вашего backend**. Скопируйте его.

---

### Шаг 4. Добавить токен и канал в Railway

Без переменных окружения бот не сможет слать сообщения в Telegram.

1. В том же сервисе откройте вкладку **「Variables」** (или **「Settings」** → **Variables**).
2. Нажмите **「Add variable」** или **「New Variable」** и добавьте по одной:

| Имя (Name)           | Значение (Value) |
|----------------------|------------------|
| `TELEGRAM_BOT_TOKEN` | `8750571471:AAHoOCjNw5f42tOJmqCXSZ-M2sHrsyxdBCs` |
| `TELEGRAM_CHAT_ID`   | `@promauto_rf` |
| `ALLOWED_ORIGIN`     | `https://psve.ru` |

3. Сохраните. Railway перезапустит приложение сам.

---

### Шаг 5. Проверить, что адрес работает

1. Откройте в браузере вашу ссылку Railway и в конце добавьте: **`/api/health`**  
   Пример: `https://helperstabur-production-xxxx.up.railway.app/api/health`
2. Должна открыться страница с текстом: `{"ok":true,"service":"helperstabur"}`.  
   Значит, **адрес сервера есть и backend отвечает**.

---

### Шаг 6. Вставить этот адрес в Tilda

В настройках сайта Tilda (код перед `</body>`) вставьте одну строку, подставив **вашу** ссылку Railway (без `/api/health`):

```html
<script src="https://ВАША-ССЫЛКА-ИЗ-RAILWAY/widget/embed.js" data-api="https://ВАША-ССЫЛКА-ИЗ-RAILWAY"></script>
```

Пример:

```html
<script src="https://helperstabur-production-xxxx.up.railway.app/widget/embed.js" data-api="https://helperstabur-production-xxxx.up.railway.app"></script>
```

Сохраните и опубликуйте сайт. Виджет на psve.ru начнёт обращаться по этому адресу — вопрос «есть ли у нас адрес сервера» будет решён: да, это та ссылка, которую выдал Railway.

---

## Кратко

- **Адреса сервера изначально нет** — он не «уже где-то есть», его нужно получить.
- **Создаётся адрес так:** выкладываете backend на хостинг (например Railway) → хостинг даёт вам ссылку → эта ссылка и есть адрес backend.
- Дальше этот адрес подставляете в Tilda в строку с `embed.js` и `data-api`, как в шаге 6 выше.

---

## Вариант без GitHub (если не хотите заводить репозиторий)

На Railway можно попробовать загрузить проект без GitHub:

1. **New Project** → **「Empty Project」** или **「Deploy from local」** (если такая кнопка есть).
2. Установите у себя **Railway CLI**:  
   https://docs.railway.app/develop/cli  
   В папке проекта выполните `railway login`, затем `railway init` и `railway up` — проект загрузится, и в панели Railway появится сервис. Дальше — как в шагах 3–6: сгенерировать домен, добавить переменные, взять ссылку и вставить в Tilda.

Если вариант с GitHub для вас проще — делайте через GitHub, как в шагах 1–6.
